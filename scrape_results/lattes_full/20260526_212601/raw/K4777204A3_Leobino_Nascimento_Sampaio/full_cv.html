Explorando Amostras Observacionais para Geração de Tráfego de Rede Malicioso por Modelos Generativos Profundos (Edital Universal CNPQ/MCTI N 44/2024)
Descrição: O uso de amostras observacionais, coletando porções de dados reais, permite a geração de tráfego sintético mais realista a um custo menor, possibilitando a criação de cenários complexos para estudos de cibersegurança. Modelos generativos profundos, como as Redes Adversárias Generativas (GANs), podem aprender a distribuição do tráfego e criar novas amostras sintéticas, úteis para avaliar cenários de ataque, como de negação de serviço (DoS). Embora haja iniciativas que utilizam GANs para desenvolver técnicas defensivas, poucos estudos abordam soluções escaláveis que garantam a privacidade. GANs podem memorizar amostras específicas, levando à extração de dados originais e à violação de privacidade, assim como a reconstrução de dados de treinamento a partir de saídas do modelo. A implementação de mecanismos de privacidade diferencial nas GANs é necessária para evitar a revelação de informações individuais, embora isso reduza a qualidade dos dados gerados. A coleta de amostras observacionais enfrenta desafios em redes IP devido à variabilidade de protocolos, aplicações criptografadas e dispositivos de encaminhamento sem estado, dificultando a instrumentação e anonimização dos dados. Este projeto busca investigar a coleta, processamento e uso de amostras reais para criar dados sintéticos para estudos de cibersegurança..
Situação: Em andamento; Natureza: Pesquisa.

Automação, Escalabilidade e Interoperabilidade na Configuração e Inicialização de Segurança de Aplicações Distribuídas da Internet das Coisas - Produtividade em Pesquisa - PQ 2025
Descrição: A maioria dos sistemas da Internet das Coisas é caracterizada pela presença de dispositivos com recursos limitados que utilizam o protocolo IP como principal mecanismo de comunicação. O IP foi originalmente desenvolvido para sistemas computacionais mais robustos e conectados a redes estruturadas, a partir de um modelo de comunicação fim a fim, orientado ao destinatário e baseado em rotas predefinidas. Essas características tornam a configuração de segurança especialmente desafiadora, sobretudo devido à dinamicidade das topologias, à conectividade intermitente e à alta mobilidade dos nós. Quando um nó é conectado a uma rede IP, serviços adicionais de configuração são necessários para viabilizar a comunicação com outros nós de forma segura. O DHCP faz a configuração dinâmica de endereços de rede, máscara de sub-rede e rota padrão. Em seguida, o DNS é utilizado para a configuração de nomes e, por fim, Autoridades Certificadoras (CA) são utilizadas, seguindo um modelo de PKI, para emissão de certificados confiáveis, utilizados na criação de canais seguros para troca de dados. A configuração de segurança por meio de serviços adicionais providos por CAs, DHCP e DNS é especialmente desafiadora em cenários de IoT por se tratar de ambientes com alta dinamicidade; utilizarem o encaminhamento IP que é sem estado; e a mobilidade dos nós que demanda uma solução de segurança simples, flexível e ágil, algo que as abordagens atuais baseadas em TLS/SSL não oferecem. Estas são algumas das limitações da Internet atual que tem motivado a busca por projetos, implementações práticas e novas direções para a descentralização da arquitetura da Internet. Neste sentido, é preciso destacar o trabalho do Grupo de Pesquisa sobre a Descentralização da Internet (DINRG) da Força-Tarefa de Pesquisa da Internet (IRTF), que tem reunido pesquisadores, profissionais e líderes da indústria para discutir as tecnologias e metodologias que impulsionam a próxima geração de sistemas descentralizados da Internet. Neste contexto, este projeto busca investigar soluções de bootstrapping de segurança mais escaláveis, distribuídas e centradas no usuário através de tecnologias distribuídas (e.g., Identidade Digital Descentralizada IDD) e arquiteturas de rede clean-slate, como as Redes de Dados Nomeados (Named Data Networking NDN). Diante de tais características, o projeto busca investigar o bootstrapping de segurança de redes em cenários dinâmicos de IoT, de forma a trazer, para tais sistemas, maior escalabilidade de configuração, flexibilidade na definição de políticas, menor dependência de serviços adicionais de aplicação e de estabelecimento de conexão.
Situação: Em andamento; Natureza: Pesquisa.

Automation of Remote Security Bootstrapping Through Named-Data Networking (AFOSR - AFRL)
Descrição: The current IP's communication model is end-to-end, recipient-oriented, and path-based. These properties are harder to apply in dynamic environments featured by intermittent connectivity, dynamic topologies, and intense node mobility. When plugged into an IP network, the device's protocol stack requires a set of further connectivity configuration procedures to communicate with other nodes, carried out by infrastructure-based network services. IoT systems, however, present new mobility and security requirements that current infrastructure-based network services fail to address due to some reasons. Firstly, in dynamic environments, there is no guarantee that these services will be available when plugging a new device into the network. Secondly, IP forwarding is stateless and has no adaptability of its own, so requiring constant protocol configurations. Thirdly, node mobility demands a simple, flexible, and fast security configuration, not seen in current TLS/SSL-based solutions. Named Data Networking (NDN)'s built-in features can meet these requirements. It is a networked system formed by named entities through which the names stem from those used by applications with semantic meaning and its communication model focuses on the content itself (i.e., what) rather than its location (i.e., where), using directly secured data signed by data producers. However, the need for name assignments and trust configuration in deploying new entities in an NDN network requires systematic solutions to minimize manual operations during NDN configurationsin IoT environments, particularly those formed by remotely-located massive devices. Hence, this project aims to investigate trust management and configuration, remote security bootstrapping, and easy-to-use solutions to support NDN deployment in mobile IoT..
