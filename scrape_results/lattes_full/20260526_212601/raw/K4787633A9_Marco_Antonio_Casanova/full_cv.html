Marco A. Casanova é Pesquisador do Instituto Tecgraf e colaborador do Programa de Pós-graduação em Informática da Pontifícia Universidade Católica do Rio de Janeiro - PUC-Rio. Formou-se em Engenharia Eletrônica pelo Instituto Militar de Engenharia (1974), e obteve o Mestrado em Ciências em Informática pela PUC-Rio (1976) e o Mestrado (1978) e Doutorado (1980) em Matemática Aplicada pela Harvard University. Foi Coordenador de Pós-Graduação (2005-2007), Diretor (2007-2011), e Professor Titular (2015-2025) do Departamento de Informática da PUC-Rio, e Coordenador Central de Planejamento e Avaliação da PUC-Rio (2012-2025). Sua pesquisa concentra-se em modelagem conceitual de bancos de dados, construção de sistemas de gerência de bancos de dados, e aplicacões de Large Language Models - LLMs. Pelo conjunto das contribuições, em julho de 2012, recebeu o Prêmio Mérito Científico da Sociedade Brasileira de Computação.

Construção de Interfaces em Linguagem Natural para Bancos de Dados baseadas em Modelos Linguísticos Massivos
Descrição: Um Modelo Linguístico Massivo (Large Language Model LLM) é um modelo de linguagem baseado em uma arquitetura de rede neural profunda com um número muito grande de parâmetros, treinado em um grande volume de dados não rotulados. Este projeto baseia-se no uso de LLMs para construir interfaces (conversacionais) de banco de dados em linguagem natural (LN). A adoção de LLMs simplifica a interpretação de consultas em LN, a geração de respostas em LN e o controle do diálogo do usuário. Mais precisamente, o projeto tem como foco dois problemas: o Problema de Texto-para-SQL e o Problema de Texto-para-SQL Conversacional. O primeiro é definido como "Dado um banco de dados D e uma sentença N em LN que descreve uma pergunta sobre D, gere uma consulta Q em SQL sobre D que expressa N". Uma ferramenta de texto-para-SQL fornece uma maneira direta para construir interfaces de banco de dados em linguagem natural. Já o segundo é definido como: "Dado um banco de dados D com um esquema S, traduza uma sequência de perguntas em LN sobre D em suas consultas SQL correspondentes." Este problema é muito mais complexo do que o anterior, uma vez que a resposta a uma pergunta pode depender do estado do diálogo, incluindo as respostas às perguntas anteriores. O projeto investiga soluções para estes dois problemas, onde o processo de tradução é majoritariamente executado por LLMs. Os resultados esperados são implementações de soluções para estes problemas, no estado-da-arte, testadas com dados reais, e que possam ser transferidas para um ambiente industrial. A ênfase em testar as soluções sobre dados reais destaca a pesquisa proposta das soluções encontradas na literatura, focadas no desempenho sobre benchmarks artificiais. Quanto à formação de recursos humanos, o projeto prevê a orientação de dois alunos de doutorado, três alunos de mestrado, e três alunos de graduação. Por fim, o projeto prevê a publicação de três artigos em periódicos e seis trabalhos em conferências..

Fundamentos de Acervos de Dados
Descrição: Um acervo de dados hospeda coleções de dados, sejam elas representadas sob forma de um arquivo convencional, um banco de dados, ou uma biblioteca digital, entre outras formas. Um acervo de dados, por definição, não possui um esquema global, ou sofre alguma forma de integração a priori dos dados, como em um armazém de dados (Data Warehouse). O projeto de pesquisa proposto para 2022/03 a 2026/02 intitula-se Fundamentos de Acervos de Dados. O projeto tem como objetivos: (1) investigar técnicas para facilitar o uso de acervos de dados; (2) investigar métodos para facilitar o desenvolvimento de aplicações de Ciência de Dados sobre acervos de dados. A pesquisa em torno do primeiro objetivo concentrar-se-á em técnicas para descrever coleções de dados, integrar sob demanda coleções de dados, e acessar um acervo de dados, e abordará a prototipação de um sistema de gerência de acervos de dados. A pesquisa em linha com o segundo objetivo endereçará a descrição de classes de aplicações de Ciência de Dados, a construção de interfaces que auxiliem o usuário a realizar tarefas típicas de projetos de Ciência de Dados sobre acervos de dados, e a implementação de aplicações-piloto de Ciência de Dados sobre acervos de dados..

Descrição: Financiadores:

FAPERJ - Edital E_26/2014 - CIENTISTA DO NOSSO ESTADO - 2014 - Proc. E-26-201.337-2014
As aplicações desenvolvidas para a Web, de visualizações simples a ferramentas interativas sofisticadas, facilitam consideravelmente o acesso a dados. Porém, tais aplicações processam dados em volumes variados, em formatos diferentes e oriundos de diversas origens. A W3C Data Activity: Building the Web of Data foi criada para endereçar os problemas gerados por esta diversidade e, portanto, potencialmente facilitar a integração e o processamento de dados publicados na Web. Esta iniciativa desenvolve recomendações, formatos, modelos e ferramentas para troca de dados. Porém, publicar e acessar dados segundo os preceitos da Web de Dados gera desafios. O objetivo geral deste projeto consiste em investigar os fundamentos, criar técnicas e desenvolver aplicações experimentais para tornar a publicação e o acesso a dados na Web de Dados tão simples quanto atualmente é a publicação e o acesso a páginas Web.No que tange a fundamentos, o projeto estudará inicialmente o problema de reuso de ontologias no projeto de esquemas RDF. Este problema será abordado através da definição de procedimentos de decisão estruturais para ontologias expressivas. Tais procedimentos permitem a definição de álgebras de ontologias que, por sua vez, formalizam a noção de reuso de vocabulários e de restrições de integridade. O projeto tratará em seguida do problema de fusão (mashup) de dados inconsistentes. Este segundo problema será abordado através da definição de procedimentos para verificação de modelos em Lógicas de Descrição com Defaults.As técnicas para construção da Web de Dados abordarão: a descoberta de relacionamentos entre entidades; a recomendação de ontologias para publicação de bancos de dados; a recomendação de conjuntos de dados a serem interligados; a manutenção de interligações materializadas entre conjuntos de dados; a manutenção de fusões materializadas de dados; a fusão de dados híbridos (dados estáticos e sequências de dados). Os resultados esperados são técnicas para publicação de bancos de dados e de sequências de dados na Web de Dados, de uma nova geração, oferecendo funcionalidade mais sofisticada. O projeto abordará o desenvolvimento de duas classes de aplicações: indução de valores de propriedades de entidades e recomendação de entidades. O projeto explorará como as características da Web de Dados podem facilitar o desenvolvimento destas classes de aplicações. Basicamente, a Web de Dados provê conjuntos de dados estruturados que facilitam o treinamento de algoritmos de aprendizado de máquina para interpretação automática de texto, simplifica a fusão de dados oriundos de fontes distintas e oferece diretrizes para a divulgação de dados de forma aberta.As contribuições esperadas são de diferentes naturezas: (1) contribuições aos fundamentos da Web de Dados; (2) desenvolvimento de novas técnicas para facilitar a publicação de bancos de dados e de sequências de dados na Web de Dados; (3) desenvolvimento de aplicações inovadoras e realistas que ilustrem as vantagens da Web de Dados; (4) formação de recursos humanos, através da inclusão de mestrandos, doutorandos e pós-doutores no projeto; (5) fortalecimento da cooperação entre os pesquisadores das instituições envolvidas no projeto; (6) publicações sobre os resultados de pesquisa para disseminação do conhecimento sobre a Web de Dados..

SEEK - SEmantic Enrichment of trajectory Knowledge discovery
Descrição: A flood of data pertinent to moving objects is available today, and will be more in the near future, particularly due to the automated collection of data from personal devices such as mobile phones and other location-aware devices. Such wealth of data, referenced both in space and time, may enable novel classes of applications of high societal and economic impact, provided that the discovery of consumable and concise knowledge out of these raw data is made possible. The fundamental hypothesis is that it is possible, in principle, to aid citizens in their mobile activities by analyzing the traces of their past activities by means of data mining techniques. For instance, behavioral patterns derived from mobile trajectories may allow inducing traffic flow information, capable to help people travelling efficiently, to help public administrations in traffic-related decision making for sustainable mobility and security management. Behavioral patterns can be extracted through a knowledge discovery process where positioning data collected from mobile devices are first transformed in semantically enriched trajectory data stored in a database. Then, these data are loaded in a data warehouse and analyzed with OLAP operations that allow summarization of the trajectories features. Mobility patterns, the most common movements emerging from data, are computed with suitable spatio-temporal data mining algorithms. A further semantic enrichment step is needed to give context-dependent meaning to the discovered patterns. The goal of the project is to investigate methods to extract meaningful knowledge from large amount of movement data by defining techniques for an advanced semantic-rich knowledge discovery process..

Descrição, Publicação e Acesso a Conjuntos de Linked Data
Descrição: Os princípios que regem a produção de Linked Data fornecem subsídios para a publicação e conexão de dados por meio da infraestrutura fornecida pela Web. Do ponto de vista do desenvolvimento de aplicativos, sugerem uma forma tanto para publicar bancos de dados na Web quanto para facilitar a interoperabilidade entre bancos de dados. Porém, o armazenamento e acesso a grandes conjuntos de triplas RDF, organizados segundo os princípios de Linked Data, representam um grande desafio. De fato, nem os repositórios especializados em triplas RDF, nem os processadores de consultas SPARQL escalam para o volume de dados desejado. Novos paradigmas devem então ser investigados para viabilizar este cenário. Computação em nuvem apresenta-se como uma opção na medida em que provê armazenamento e poder de processamento em larga escala e sob demanda. O objetivo geral do projeto consiste em desenvolver métodos, ferramentas e experimentos envolvendo a descrição, publicação e acesso a conjuntos de Linked Data. No que tange a descrição de conjuntos de Linked Data, o projeto enfatiza a qualidade do conjunto final de triplas RDF, medida tanto pela facilidade em identificar a semântica das classes e propriedades incluídas na descrição do conjunto de triplas RDF, via as restrições de integridade da ontologia de aplicação, quanto pela estratégia adotada para gerar as URIs que identificam os recursos descritos.No contexto de publicação de conjuntos de Linked Data, o projeto expande a estratégia de triplificação tradicional para também publicar triplas RDF em páginas Web estáticas com RDFa embutido. Explora também o uso de plataformas de computação em nuvem como opção para armazenamento e acesso a grandes grafos RDF, enfatizando técnicas de indexação.Quanto a acesso a conjuntos de Linked Data, o projeto explora métodos para otimização de consultas SPARQL no contexto de processamento exploratório e no contexto de processamento mediado..

Métodos e Técnicas para Alinhamento de Esquemas Conceituais
Descrição: Alinhar dois esquemas conceituais consiste em definir um relacionamento entre os conceitos dos dois esquemas de tal forma que, intuitivamente, conceitos relacionados possuem o mesmo significado. Alinhamento de esquemas conceituais é um problema fundamental na mediação de consultas e na criação de armazéns de dados. Um primeiro enfoque para alinhamento de esquemas, chamado de enfoque sintático, consiste em alinhar conceitos tomando como base sugestões sintáticas, como os nomes e tipos de dados das propriedades. Este enfoque depende da suposição implícita de que proximidade sintática implica em proximidade semântica. O segundo enfoque, chamado de enfoque semântico (extensional ou baseado em instâncias), utiliza as próprias instâncias armazenadas nos bancos de dados para gerar hipóteses sobre o alinhamento dos conceitos. Este enfoque é mais robusto do que o enfoque sintático, mas aplica-se apenas a esquemas sem grande complexidade estrutural. Tanto o enfoque sintático quanto o enfoque semântico operam a posteriori, no sentido de que tentam alinhar esquemas conceituais pré-existentes, que tipicamente não foram projetados para tal. Contrastando com esta situação, podemos definir o enfoque a priori, baseado na premissa de que, se o banco de dados deverá ser integrado com outros bancos de dados (através de um mediador ou de um armazém de dados), então o seu esquema conceitual (ou o esquema exportado) deverá ser baseado em um esquema conceitual de referência para a área de aplicação a que o banco se refere. Tendo em vista que o enfoque sintático foi largamente explorado, este projeto concentra-se no desenvolvimento de técnicas, seguindo o enfoque semântico a posteriori, para alinhamento de taxonomias utilizadas na classificação de objetos em bancos de dados distintos e para alinhamento de esquemas conceituais em geral. O projeto investiga ainda métodos para criação de esquemas conceituais de referência através do alinhamento e reuso de fragmentos de ontologias..

Infraestrutura de Metadados para Objetos Científicos
Descrição: Em diversas áreas do conhecimento, investigação científica hoje depende da estreita interação entre grupos de pesquisa geograficamente dispersos, detentores de dados, processos ou tecnologia complementares entre si. Esta forma de investigação necessita portanto de uma infra-estrutura de serviços computacionais que permita um acesso fácil, seguro e coordenado a dados e processos geograficamente distribuídos. Dados científicos são provenientes de fontes tão diversas quanto aceleradores de partículas, sensores a bordo de satélites, seqüenciamento de genomas, por exemplo, ou derivados de outros dados através de processos de análise. São são representados de várias formas ? estruturada, semi-estruturada e não-estruturadas ? e estão armazenados em bancos de dados especializados, repositórios institucionais, ou mesmo nos computadores pessoais dos pesquisadores. Os processos de análise são tipicamente combinados em processos mais complexos, modelados como workflows. As tarefas de localizar, avaliar a utilidade, extrair, integrar ou combinar estes dados e processos requerem técnicas e ferramentas específicas, endereçando desde gerência de metadados até métodos de otimização para execução de workflows, em um ambiente distribuído de computação. Além disto, tais técnicas e ferramentas não devem ser específicas para um particular domínio de aplicação. Todas as peculiaridades do domínio devem ser isoladas em uma descrição formal, que as ferramentas interpretam para realizar o seu trabalho. Neste contexto, este projeto destina-se a: 1) Desenvolver métodos, técnicas, ferramentas e software frameworks para catalogação semi-automática de metadados de objetos científicos. 2) Desenvolver métodos, técnicas, ferramentas e software frameworks para mediação de acesso aos metadados de objetos científicos. 3) Desenvolver métodos, técnicas, e software frameworks para compartilhamento de resultados entre comunidades de usuários trabalhando em experimentos virtuais..

Cooperativas de Fontes de Dados
Descrição: Este projeto visa desenvolver métodos e ferramentas para criação de cooperativas de fontes de dados, definidas como conjuntos de fontes de dados autônomas, interligadas através de mediadores que oferecem consulta e atualização dos dados. Uma fonte de dados é um termo neutro para designar um conjunto de dados encapsulados por um sistema de gerência de banco de dados ou por uma aplicação especificamente desenvolvida para tal. Os métodos de projeto propostos para cooperativas de fontes de dados enfatizarão a definição de padrões estendíveis para organização e acesso aos dados, simplificando a criação de mediadores. As ferramentas de consulta propostas incluirão mecanismos de flexibilização que substituem o conceito de resposta exata pelo conceito de resposta pelo melhor esforço. As ferramentas de atualização propostas também incluirão mecanismos para processamento flexível de workflows, resgatando a noção de transação, na medida do possível. Os métodos e ferramentas propostos manterão a liberdade das fontes de dados, inerente à noção de cooperativa. Os métodos e ferramentas propostos terão impacto tanto no contexto aberto da Web, expondo a semântica das fontes de dados de forma processável por software, quanto no contexto de corporações cujos dados residem em uma variedades de fontes relativamente autônomas. O primeiro contexto corresponde à visão da Web semântica, com a diferença de que a semântica tornada visível é a do conjunto de dados armazenados em uma fonte de dados. O segundo contexto é essencialmente o de uma federação de bancos de dados, exceto que elimina à priori os problemas oriundos da heterogeneidade semântica dos dados..
